# DW Migration Assistant

A modern web application built with React, TypeScript, Material-UI, and Framer Motion to help simplify data warehouse migrations to Databricks SQL.

## Features

- **Multi-Platform Support**: Supports migrations from 7 major data warehouse platforms:
  - Oracle Data Warehouse
  - Snowflake
  - Microsoft SQL Server
  - Teradata
  - IBM Netezza
  - Azure Synapse Analytics
  - Amazon Redshift

- **Comprehensive Data Type Mappings**: View detailed mappings for 200+ data types from source systems to Databricks SQL
- **Interactive UI**: Smooth animations and intuitive navigation powered by Framer Motion
- **Search Functionality**: Quickly find specific data type conversions
- **Migration History**: Track your past migration queries and activities
- **Analytics Dashboard**: View insights and statistics about your migrations
- **Responsive Design**: Works seamlessly on desktop and mobile devices

## Technology Stack

- **React 18** with TypeScript
- **Material-UI (MUI)** for component library
- **Framer Motion** for animations
- **Emotion** for styling

## Getting Started

### Prerequisites

- Node.js (version 14 or higher)
- npm or yarn

### Installation

1. Clone the repository:
```bash
git clone https://github.com/suryasai87/dw_migration_app.git
cd dw_migration_app
```

2. Install dependencies:
```bash
npm install
```

3. Start the development server:
```bash
npm start
```

4. Open [http://localhost:3000](http://localhost:3000) to view it in the browser.

## Available Scripts

- `npm start` - Runs the app in development mode
- `npm test` - Launches the test runner
- `npm run build` - Builds the app for production

## Usage

1. **Navigate to Data Type Mappings**: Click on "Data Type Mappings" in the sidebar
2. **Select Source System**: Choose your source data warehouse from the dropdown
3. **Review Mappings**: Browse through the comprehensive data type conversion table
4. **Search**: Use the search bar to find specific data types
5. **Track History**: View your migration activities in the Query History section
6. **Analyze**: Check the Analytics dashboard for insights and statistics
